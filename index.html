Differences between SVG and other web image formats

jpeg, png, gif SVG
Raster (resolution dependent) Vector (resolution independent)
Binary Test-based, XML markup, human readable-ish and human writable-ish
Elements within the image not selectable in browser Selectable in browser (drawn by browser, can be changed in browser!)
  • Most developers know that SVG is the vector image format for the web
  • There are other important differences between SVG and raster images that give us opportunities to do cool stuff in the browser and in Drupal Themes.
A quick note: SVG was bown in 1998, so it may be older than some folks in this room. It did take a long time for browsers to fully support SVG, and I often encounter doubts about browser support for SVG, but as you can see here in this recent screenshot from caniuse, unless you need to support IE 8 and 9 you are good to go. In my anecdotal experience I have had infinitely more cross-browser problems with flexbox than I ever have with SVG.

Using SVG: Easy Mode

							
<img src="picture.svg">
  • This is NOT inline. This is the simplest way to use SVG on your site: include it an image tag, just as you would with a raster image
  • This gives you the basic benefits of SVG--it's vector so it looks sharp on every screen at any size, and it tends to have a smaller filesize than raster graphics.

Twig Includes

							
{% include (‘path/to/file/picture.svg’) %}

↑ points to docroot

  • So my first tip for using inline SVG in drupal 8 is to take advantage of TWIG's include function.
  • Include will print the SVG inline in the rendered HTML, but you won't have to look at it while theming.
  • By default points to docroot
  • which is almost certainly not where you want to store your SVG files
							
{% include (‘@theme_name/picture.svg’) %}

↑ points to theme’s templates folder

  • Use the theme namespace to point to theme's templates folder
							
{% include (‘@theme_name/../images/picture.svg’) %}

↑ points to theme’s images folder

  • Hop up a step to point to theme's images folder.
  • Later on I’ll discuss using twig within the svg markup, so I’ll just mention now that files included this way will be rendered, so you can include twig functions and statements in your SVG and still use the include to pull them into your template.

This is the same template I showed you before, using include. Much easier to look at.

Anatomy of an SVG

  • Now that I've shown you how not to have to look at SVG markup...let's look at some SVG markup.
  • And while we are in here keep an eye out for opportunities to customize with values from the database
  • This is a PNG.
  • Gives you a clear message that you should not try to edit this by hand.
  • What you see when you open SVG in code editor is much friendlier.
  • On the left is code, on the right is the SVG drawn by the browser.
  • Going to walk through the code a bit.
  • SVG Root element with viewbox attribute.
  • The viewBox is an attribute of svg that determines the coordinate system and aspect ratio of the SVG. Viewbox defines the canvas that elements within the SVG are drawn on. The four values you see there are min x, min y, width or max X, and height or max y. Coordinates within the SVG are relative to the viewbox. This is totally independent of the width and height of the SVG on the page.
  • Next up we start in on the paths and shapes that make up our smiley face image. To make an analogy to html, these elements are “absolutely positioned” within the SVG’s viewbox.
  • like absolutely positioned elements in HTML, Items that are defined at the top of the file, like the yellow circle, appear behind elements that are defined below them, like the eyes and mouth. (MOVE ONE AND DEMONSTRATE)
  • FIRST element is the yellow circle. it has x and y coordinates and a radius. (ADJUST ONE)
  • The G tag is for grouping elements inside of it so you can manipulate and style them all at once. I point it out because you’ll notice that tags within the SVG can have classes and IDs just like HTML elements do.
  • Path: The path is the mouth in this example. The part beginning with D is the path definition. In theory you can decipher this and edit this by hand but I usually leave that part up to Sketch or Illustrator, or a javascript animation library.
  • SVG conveniently uses HTML/CSS syntax for styling, but the properties, like Fill and Stroke, are unique to SVG.

Color, Pattern, or Gradient, same as Fill values above

stroke=”green”

Default: 1

A number, in viewport units or a % of viewport

stroke-width=”5”

stroke-width=”20%”
  • As we saw in the smiley face example, paths in SVG have properties and attributes that are the same or similar to CSS properties.
  • These are really low-hanging fruit for setting with field data in twig.
  • With the colorfield module you can set fill and stroke
  • Opacity and stroke-width take simple number values.

SVG Filters

  • You can achieve many photoshop-like effects with SVG filters
  • some shown here include a duotone effect and a sepia effect
  • The key being, with SVG, that the filters are applied to dynamic content—your users can upload a normal profile picture, you can apply effects to it, and then down the road when you decide maybe that your sepia-tone thing is played out, you can remove that filter in your theme without worrying about images with hardcoded effects on them in the database.

SVG Filters

						
<filter id="lightenAndSaturate">
 <feColorMatrix type="matrix" result="lightened"
  values=" 2   0   0   0   0
           0   2   0   0   0
           0   0   2   0   0
           0   0   0   2   0 "/>
 <feColorMatrix type="saturate" result="saturated" values="1.3" />
</filter>
  • Here is the code. The filter SVG element is a container for Filter Effects--the FE here stands for filter effect. The filter tag has an ID on it which you use to refer to the filter, either elsewhere in your SVG or in your CSS.
  • You do not need to memorize the syntax of the filter effects--I certainly haven't, I look at the mozilla documentation every time. This one is lightening and then saturating. 
							
filter: url(#lightenAndSaturate);
  • This is how you would apply the filter to an element from your CSS
  • The SVG does not need to be inline to use filter effects, providing the path to the file in the CSS will work.
  • In example I am am about to show you I am using a filter from an inline SVG.

SVG Text and Raster Images

So far, we have been doing simple conditionals and passing color values to our inline SVG. But SVG can also contain text and raster images, and those just happen to be pretty common types of data fields in drupal. We're going to piece together an eye-catching heading by combining filters, images, and text in an SVG.

Text in SVG

						
<text x="50%" y="50%" font-size="300px" text-anchor="middle">
Here’s some vertically-centered text!</text>
  • Text in SVG is really very cool to use for heavily designed text like page-headings. Text in SVG has attributes that text in html does not.

Animated text fill by Cesar C

  • Unlike text baked into raster images, SVG text is selectable and can be read by screenreaders.

SVG Text Mask w/ Video Fill by Dudley Storey

  • You would only want to use text in SVG for short passages like names and other labels that will fit within one line
  • because although text in SVG scales beautifully, it cannot do line-wrapping!
  • When you include your SVG inline in a template, your SVG can use the same fonts as the rest of your webpage.

    • Raster Images in SVG

    • You can also include raster images in your SVG using SVG’s IMAGE tag.
    • images will remain raster, but they scale with your SVG and are handy to be manipulated with some cool SVG filters, clip-masks and effects.
    • This is the syntax you would use to include an image in your SVG--notice that the image tag is actually spelled-out “IMAGE” rather than IMG.
